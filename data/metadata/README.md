# Metadata Layout

Only JSON files in `production/` are canonical runtime metadata.

## Directories

- `production/`
  - `real_behavior_metadata.json`: canonical metadata for `behavior_gen`.
  - `real_metadata.json`: canonical metadata for the core TMT factor framework.
  - `all_industry_metadata.json`: canonical all-industry metadata.
- `fixtures/`
  - `mock_behavior_metadata.json`: behavior framework test fixture.
  - `mock_tmt_metadata.json`: core framework test fixture.
- `configs/`
  - `real_panel_config_template.json`: template for building the real TMT panel.
- `generated/`
  - Intermediate or candidate metadata that can be regenerated by scripts.
- `archive/behavior/`
  - Historical behavior metadata snapshots and compatibility copies.
- `archive/core/`
  - Historical core metadata variants.

## Rules

1. Runtime code must read from `production/`.
2. Tests and mock examples must read from `fixtures/`.
3. Builders may write candidates to `generated/`, but promoted output must be
   written explicitly to `production/`.
4. Historical files belong under `archive/` and must not be used as defaults.
5. Do not place backup files directly in `data/metadata/`.
